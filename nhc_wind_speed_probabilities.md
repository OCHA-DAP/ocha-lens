# NHC Wind Speed Probability Products

## Overview

NHC publishes basin-wide (not per-storm) wind speed probability fields for 34, 50, and 64 kt thresholds, updated every 6 hours (00Z, 06Z, 12Z, 18Z) during and outside of hurricane season. Two product families are available: **GRIB2** (gridded) and **GIS shapefiles** (vectorised). Both cover the same underlying model output but differ in format, resolution, and archive depth.

The probability fields are generated by the [Monte Carlo Probability (MCP) model](https://www.nhc.noaa.gov/pdf/2009waf_wsp.pdf), which runs 1,000 track/intensity/size realisations on a **0.5° lat/lon grid** (~55 km, 519×119 points, 1–60°N, 100°E–1°W). The 5 km products are derived from this grid by bilinear interpolation.

## GRIB2 — Gridded fields

| | |
|---|---|
| **Archive URL** | `https://ftp.nhc.ncep.noaa.gov/wsp/YYYY/MM/tpcprblty.YYYYMMDDHH.grib2.gz` |
| **Operational URL** | `https://ftp.nhc.ncep.noaa.gov/wsp/download/WSPtpcprblty.YYYYMMDDHH.gXXX.2.gribpreliminary` |
| **Archive availability** | 2006-06-15 18Z — present (GRIB1 only before this date, from 2005-06) |
| **Update cadence** | Every 6 hours; preliminary file posted ~1 hr after cycle, moved to archive ~1 hr later |
| **Contents** | 186 messages: 3 thresholds × 42 step ranges (cumulative 0–Xhr + 6-hr incremental) + 60 wind vector component messages |
| **Values** | Raw float probability, 0–100% |
| **Key GRIB2 fields** | `shortName=10si`, `stepRange=0-120`, `scaledValueOfUpperLimit` encodes threshold (34 kt = 17491, 50 kt = 25722, 64 kt = 32924) |

The operational preliminary file is overwritten each cycle. The archive retains all historical files with no observed gaps.

## GIS Shapefiles — Vectorised polygons

Three shapefile products exist, with different archive depths and schemas:

| Product | Archive URL pattern | Availability | Schema |
|---|---|---|---|
| **5 km** (primary) | `nhc.noaa.gov/gis/forecast/archive/YYYYMMDDHH_wsp_120hr5km.zip` | 2017-04-19 — 2025-10-31 | Pre-binned polygons, `PERCENTAGE` string label |
| **halfDeg** | `…_wsp_120hrhalfDeg.zip` | 2009-05-28 — 2025-10-31 | Raw float points, `PWIND120` column |
| **tenthDeg** | `…_wsp_120hrtenthDeg.zip` | 2009-05-28 — 2017-05-11 | Raw float polygons, `PWIND120` column (discontinued) |

Each zip contains **3 shapefiles** (one per wind threshold). The 5 km product has 11 rows per shapefile — one pre-binned polygon per NHC probability class (`<5%` through `>90%`). The [GIS archive index](https://www.nhc.noaa.gov/gis/forecast/archive/) can be scraped to enumerate all available files. **Note**: the GIS archive stopped updating after 2025-10-31; the GRIB2 FTP remains current.

## Linking wind fields to specific storms

Neither product encodes an ATCF storm ID — both are basin-wide composites. Three strategies for attribution:

1. **Issuance timestamp join** — `CurrentStorms.json` provides `windSpeedProbabilitiesGIS.issuance` per storm alongside the storm's ATCF ID. At any issuance time, all active storms share the same basin-wide field. Store a `(atcf_id, issuance)` link table populated at fetch time.

2. **FTP GRIB2 per-storm products** — the `/toa/` directory (`ftp.nhc.ncep.noaa.gov/toa/`) has per-storm GRIB2 files (`AL082023_TOA_TOD_34kt_advXXX.grib2`) but these are **Time-of-Arrival/Dissipation** products, not wind speed probabilities.

3. **Spatial intersection against track cone** — intersect the basin-wide probability polygons with each storm's forecast cone to approximate per-storm attribution. This is not how NHC produces the field and will misattribute overlapping storms, but may be acceptable for single-storm seasons.
